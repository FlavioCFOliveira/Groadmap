## Executive Summary

Groadmap v1.13.1 is a maintenance release for the `rmp` roadmap-management CLI.
It refreshes third-party dependencies and remediates every finding from the
2026-06-29 release-readiness audit.

The user-facing fix restores correct parsing of Cypher queries passed through the
`--query` flag when the value begins with a negative numeric literal. Values such
as `-1` or `-0.5` were previously misclassified as a missing flag value and
rejected; they are now accepted, while genuine flag forms (`--x`, `-<letter>`)
continue to be treated as flags. The release also upgrades the embedded GoGraph
knowledge-graph engine and the SQLite driver, and reconciles the GoGraph version
pin across the specification so it matches the module the project builds against.
Internal hardening completes the cycle: the `gosec` security gate is now green,
the `internal/cypherguard` package reaches full unit-test coverage, and a new
end-to-end test pins the `rmp audit stats` JSON key set against regressions.

Under Semantic Versioning 2.0.0 this is a **PATCH** release: it contains a
backward-compatible bug fix and dependency maintenance only. No `rmp` command,
flag, or JSON success schema is added, removed, or renamed; the GoGraph upgrade
is additive with no exported API break and no on-disk store migration; and the
database schema version remains `1.8.0`.

## Changes by Category

### Fixed — `--query` Values Beginning With a Negative Number

The `rmp graph query` / `search` family of commands now accepts a `--query`
value that starts with a negative numeric literal, such as `-1` or `-0.5`. Such
values were previously misclassified as a missing flag value and rejected,
because any argument starting with `-` was treated as the beginning of a flag.
The refined rule treats only genuine flag forms (`--x` and `-<letter>`) as flags,
so a leading negative number is correctly consumed as the flag's value. The
contract is documented in `SPEC/GRAPH.md`, "Cypher Input Source and Precedence",
rule 4.

### Changed — Dependency Refresh

- **GoGraph upgraded to `v0.6.0`.** The embedded GoGraph knowledge-graph engine
  moves from `v0.4.0` to `v0.6.0`. The upgrade is a pre-1.0 additive change: no
  exported Go API is broken and the on-disk graph store requires no migration.
- **SQLite driver refreshed.** `modernc.org/sqlite` moves from `v1.52.0` to
  `v1.53.0` and its `modernc.org/libc` runtime from `v1.73.4` to `v1.73.5`.

### Changed — Specification Reconciliation

- **GoGraph pin reconciled to `v0.6.0`.** The GoGraph version pin recorded in
  `SPEC/BUILD.md`, `SPEC/GRAPH.md`, and `SPEC/ARCHITECTURE.md` is reconciled from
  `v0.3.2` to `v0.6.0`, so the specification matches the module the project
  actually builds against.

### Internal

- **Security gate green.** The `gosec` security scan now reports zero issues; the
  eight verified-safe findings raised during the audit are annotated with
  `#nosec` and justified in place. `make check` runs the security gate as part of
  the standard validation battery.
- **`internal/cypherguard` fully covered.** New unit tests raise the package from
  0 % to 100 % statement coverage.
- **Audit-stats key regression test.** A new end-to-end test
  (`tests/test_45_audit_stats_keys.py`) pins the JSON key set returned by
  `rmp audit stats`, guarding the command's output contract against accidental
  change.
