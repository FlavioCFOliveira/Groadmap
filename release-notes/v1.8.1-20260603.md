# Release Notes — Groadmap v1.8.1

**Date:** 2026-06-03
**Type:** Patch (bug fix)
**Tag:** `v1.8.1`
**Previous release:** `v1.8.0` (2026-06-03)

---

## Executive Summary

Groadmap v1.8.1 fixes a rendering defect in the `rmp web` knowledge-graph page
that shipped with the read-only web interface in `v1.8.0`. On the
`/roadmaps/{name}/graph` page, the empty-state overlay was painted on top of a
populated knowledge graph, hiding the graph from view even though it had
rendered correctly underneath. The interface therefore appeared to show "no
graph" for roadmaps that did, in fact, have one.

This release restores the behaviour already specified in `SPEC/WEB.md`
(§ Empty graph): the empty-graph state appears only when the graph is genuinely
empty. It introduces no new features and changes no API, exit code, JSON output
schema, or database schema. Under Semantic Versioning this is a **PATCH**
release, fully backward compatible with `v1.8.0`. Existing installations require
no migration.

---

## Highlights

- The `rmp web` knowledge-graph page now displays the graph instead of the
  empty-state overlay.
- Root cause and fix are CSS-specificity related: a global
  `[hidden] { display: none !important; }` reset now guarantees the `hidden`
  attribute wins over component `display` rules.
- A new regression test pins the fix in place so it cannot silently return.
- No API, exit-code, JSON output, or database schema changes; schema version
  unchanged at `1.6.0`.

## Changes by Category

### Fixed — `rmp web` Graph Page Rendered the Empty-State Overlay Over a Populated Graph

The knowledge-graph page at `/roadmaps/{name}/graph` rendered the empty-state
overlay on top of the rendered graph, hiding it.

**Symptom.** Opening the graph page for a roadmap that had knowledge-graph data
showed the empty-graph message and a blank canvas, as though the roadmap had no
graph. The graph was never actually missing.

**Root cause.** A CSS-specificity conflict. The component rule
`.graph-empty { display: flex }` (a class selector) outranked the user-agent
rule `[hidden] { display: none }` (an attribute selector) on specificity.
Because the empty-state overlay element was styled with `position: absolute;`,
`inset: 0;`, and an opaque background, applying the `hidden` attribute to it did
not remove it from the layout: the `display: flex` class rule still won, so the
overlay stayed painted on top of the Cytoscape canvas. The graph itself
initialised correctly underneath the overlay; it was simply covered.

**Fix.** A global reset rule was added to `internal/web/static/style.css`:

```css
/* The `hidden` attribute must always win over component display rules. */
[hidden] { display: none !important; }
```

With `!important`, the `hidden` attribute reliably overrides any component
`display` rule, so a hidden overlay is removed from the layout. The empty-graph
state now appears only when the graph is genuinely empty, exactly as specified
in `SPEC/WEB.md` (§ Empty graph).

**Scope.** The change touches only the embedded stylesheet and a test file. No
Go runtime logic, route, JSON output, exit code, or database schema changed.

## Upgrade Notes

No migration steps are required. The fix is contained in the embedded web
assets; the database schema version is unchanged at `1.6.0`.

```sh
# Build from source
git fetch --tags
git checkout v1.8.1
go build -o ./bin/rmp ./cmd/rmp

# Verify
./bin/rmp --version
# Groadmap version 1.8.1

# Launch the read-only web interface and open a roadmap's graph page
./bin/rmp web
```
