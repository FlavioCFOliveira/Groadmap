## Executive Summary

v1.6.0 is a pure maintenance release that keeps the Groadmap toolchain and
CI infrastructure current. It raises the Go toolchain directive in `go.mod`
from `1.26` to `1.26.2`, upgrades seven GitHub Actions to their latest patch
releases, and bumps four transitive Go module dependencies. No new features
are introduced and no public API, exit code, or JSON output schema is altered.
Existing `v1.5.0` installations require no migration.

---

## Changes by Category

### Go Toolchain

| Item | Before | After |
|------|--------|-------|
| `go` directive in `go.mod` | `go 1.26` | `go 1.26.2` |

Raising the directive to the latest patch release ensures that CI and local
builds both use the Go standard library bugfixes shipped in 1.26.2. No source
code changes were required; only the `go.mod` directive and the corresponding
`go.sum` entries were updated.

### GitHub Actions

All action versions have been pinned to their latest patch release. This
closes any dependency-confusion exposure and aligns the CI pipeline with
upstream security patches.

| Action | Before | After |
|--------|--------|-------|
| `actions/checkout` | v6 | v6.0.2 |
| `actions/setup-go` | v6.3.0 | v6.4.0 |
| `actions/upload-artifact` | v7 | v7.0.1 |
| `actions/download-artifact` | v7 / v8 | v8.0.1 (unified) |
| `golangci/golangci-lint-action` | v8 | v9.2.1 |
| `codecov/codecov-action` | v5.5.3 | v6.0.1 |
| `softprops/action-gh-release` | v2.6.1 | v3.0.0 |

`actions/download-artifact` was previously referenced at two different versions
across `ci.yml` and `release.yml`; both references are now unified at v8.0.1.

### Go Dependencies

The following transitive Go module dependencies were upgraded. All are indirect
dependencies pulled in by GoGraph or the SQLite driver. Direct dependencies are
unchanged.

| Module | Before | After |
|--------|--------|-------|
| `google/go-cmp` | v0.6.0 | v0.7.0 |
| `golang.org/x/text` | v0.22.0 | v0.37.0 |
| `modernc.org/cc/v4` | v4.28.2 | v4.28.4 |
| `modernc.org/ccgo/v4` | v4.34.2 | v4.34.4 |

Note: the remaining indirect dependencies pinned by GoGraph (`cucumber/godog`,
`gofrs/uuid`, `hashicorp/*`, `spf13/pflag`) cannot be upgraded independently
without a new GoGraph release and are left at their current versions.

### CI / Release Workflow â€” Stale Windows References Removed

The `dev-release` step in `release.yml` contained download links for Windows
binaries. Windows builds were dropped in a prior commit because GoGraph's use
of `syscall.Kill` makes cross-compilation to Windows infeasible. The stale
references have been removed to prevent confusion in CI run summaries and
release artefact listings.

## Known Issues

The two SPEC-vs-code divergences carried forward from prior releases remain
open and are unaffected by this maintenance release:

1. **Exit-code mapping for `ErrInvalidInput`** â€” `SPEC/ARCHITECTURE.md`
   documents exit code `2`; the implementation uses `ExitInvalidData = 6`.
2. **`audit stats` JSON keys** â€” `SPEC/COMMANDS.md` documents one set of
   keys; the implementation emits a different (stable) set.

Neither divergence affects runtime behaviour. Both are tracked as `spec` /
`tech-debt` GitHub issues and will be resolved by a `specification-manager`
pass in a future release.

No E2E tests cover `rmp graph` subcommands. Coverage will be added in a
follow-up release.

## Upgrade Instructions

No migration steps are required. Replace the `rmp` binary with the v1.6.0
build. All existing roadmap databases, `~/.roadmaps/` layouts, and JSON
consumers remain fully compatible.

```sh
# Build from source
git fetch --tags
git checkout v1.6.0
go build -o ./bin/rmp ./cmd/rmp

# Verify
./bin/rmp --version
# rmp version 1.6.0
```
