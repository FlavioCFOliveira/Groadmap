# Release Notes — Groadmap v1.13.0

**Date:** 2026-06-19
**Type:** Minor (read-only Audit Log web page + Tabler-fidelity pass)
**Tag:** `v1.13.0`
**Previous release:** `v1.12.0` (2026-06-17)

---

## Executive Summary

Groadmap v1.13.0 is a web-focused minor release for the `rmp` roadmap-management
CLI. It adds a read-only Roadmap Audit Log page to the `rmp web` interface and
completes a Tabler-fidelity pass across every served template.

The new Audit Log page surfaces the roadmap's audit trail directly in the
browser, paginated with a numbered Tabler pagination control so large trails
remain navigable. In parallel, the fidelity pass aligns the entire web UI with
the Tabler design system: status, priority, and severity values are now rendered
as semantically colour-coded badges; sprint tabs adopt the Tabler
`card-header-tabs` pattern; all inline styles are removed in favour of Tabler and
the project's `style.css`; and a set of minor markup corrections bring the
templates closer to Tabler conventions. The web specification (`SPEC/WEB.md`) is
extended with explicit Tabler-fidelity rules for templates.

Under Semantic Versioning 2.0.0 this is a **MINOR** release: it adds
backward-compatible functionality (a new read-only page) and refines the existing
read-only presentation. No JSON success schema is removed or renamed, the on-disk
graph format is unchanged, and the database schema version remains `1.8.0`.

## Changes by Category

### Added — Read-Only Roadmap Audit Log Page

The read-only `rmp web` server gains a new page that exposes the roadmap's audit
trail in the browser. The page is served from the embedded templates and reads
through the same strictly read-only data path as the rest of the web UI, opening
each database with SQLite `query_only`. Entries are paginated with a **numbered
Tabler pagination** control, so large audit trails are navigable page by page.
The page is specified in `SPEC/WEB.md`.

### Changed — Tabler-Fidelity Pass Across the Web Templates

Every served template was reviewed and brought into alignment with the Tabler
design system:

- **Semantic colour-coded badges.** Status, priority, and severity values are now
  rendered as semantically colour-coded Tabler badges, making state and
  importance visually distinguishable across the served pages.
- **Sprint tabs on the `card-header-tabs` pattern.** The sprint tabs adopt
  Tabler's `card-header-tabs` markup, aligning the sprint views with the design
  system.
- **Inline styles removed.** All inline `style` attributes were removed from the
  templates; presentation is now driven exclusively by Tabler and the project's
  `style.css`.
- **Minor markup fidelity.** Several small corrections bring the templates closer
  to Tabler conventions, including grid gap utilities (`g-2`), an `h1` brand
  heading, and footer markup.

### Changed — Specification and Documentation

- **`SPEC/WEB.md`** is extended with explicit Tabler-fidelity rules governing the
  web templates.
- The web documentation notes the semantic colour-coded status/priority/severity
  badges.

### Internal

- New web unit suites covering the Audit Log page (`internal/web/audit_test.go`),
  semantic badges (`internal/web/badge_test.go`), pagination
  (`internal/web/pagination_test.go`), the no-inline-style invariant
  (`internal/web/inline_style_test.go`), Tabler-fidelity markup
  (`internal/web/tabler_fidelity_test.go`), and sprint rendering
  (`internal/web/sprint_test.go`).
- New audit-log query support in `internal/db/queries.go` with accompanying unit
  tests (`internal/db/queries_test.go`).
- Extended web end-to-end coverage in `tests/test_35_web_interface.py`.

None. This release adds a new read-only web page and refines the presentation of

removed or renamed, the on-disk graph format is unchanged, and the database

## Upgrade Notes

No manual migration steps are required. This release adds no SQLite migration of
its own and the database schema version is unchanged.

```sh
# Build from source
git fetch --tags
git checkout v1.13.0
go build -o ./bin/rmp ./cmd/rmp

# Verify
./bin/rmp --version
# Groadmap version 1.13.0

./bin/rmp --ai-help | grep binary_version
#   "binary_version": "1.13.0",
```

The new Audit Log page is reached from the `rmp web` interface; start the
read-only server as usual and navigate to the audit log from the web UI.
