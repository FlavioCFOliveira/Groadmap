# Release Notes — Groadmap v1.12.0

**Date:** 2026-06-17
**Type:** Minor (web interface enhancements + help and AI-contract review)
**Tag:** `v1.12.0`
**Previous release:** `v1.11.0` (2026-06-16)

---

## Executive Summary

Groadmap v1.12.0 is a combined minor release for the `rmp` roadmap-management
CLI. It pairs read-only `rmp web` interface enhancements with a full review of
the command-line help surface and the machine-readable AI Agent Contract emitted
by `rmp --ai-help`.

On the web side, the graph view gains an interactive query bar, a clickable
labels sidebar with highlight, neighbour-focus navigation, and unified sprint
cards; the read-only pages now also surface each sprint's title and execution
order. On the contract side, every plain-text help printer was revised for
correctness and completeness, the AI Agent Contract was normalised for internal
consistency, and a set of documented-but-divergent exit codes was corrected so
that the runtime now matches the published contract. The `GoGraph` dependency is
bumped to `v0.4.0`.

Under Semantic Versioning 2.0.0 this is a **MINOR** release: it adds
backward-compatible functionality and corrects exit codes to the values the
contract and help already promised, restoring rather than breaking the documented
behaviour. No JSON success schema is removed or renamed, and the database schema
version is unchanged.

## Highlights

- **Richer read-only graph view:** the `rmp web` graph page gains a query bar, a
  labels sidebar with click-to-highlight, and neighbour-focus navigation.
- **Unified sprint cards:** sprint presentation is consolidated into a single
  consistent card layout across the web interface.
- **Sprint title and execution order in the UI:** the read-only pages now display
  each sprint's title and its execution order, matching the CLI ordering.
- **Help reviewed end to end:** every plain-text help printer was corrected for
  accuracy, formatting, and completeness.
- **AI Agent Contract normalised:** nested single-action commands, `[]` instead of
  `null` arrays, min-only ranges without a misleading `max: 0`, `--max-tasks`
  `1-10000`, the `roadmap_flag` web exemption, `sprint tasks -s/--status`, and
  failure examples for commands with non-zero exit codes.
- **Exit codes corrected to the contract:** invalid `--type`/`--status` on list
  commands now exit 6; `sprint bottom` on a missing sprint exits 4; `audit stats`
  emits `null` for empty-set timestamps.
- **Dependency refresh:** `GoGraph` bumped to `v0.4.0`.

## Changes by Category

### Added — Web Interface Enhancements

The read-only `rmp web` server gains several navigation and presentation
features in its knowledge-graph and sprint views:

- **Graph query bar** for filtering the visualised graph interactively.
- **Labels sidebar** with click-to-highlight, so selecting a label highlights the
  matching nodes.
- **Neighbour focus**, letting the reader centre the view on a node and its
  immediate neighbours.
- **Unified sprint cards**, consolidating sprint presentation into one consistent
  layout.
- **Sprint title and execution order** surfaced in the read-only UI, so the served
  pages reflect the same ordering the CLI uses.

These features are specified in `SPEC/WEB.md` and remain strictly read-only: the
per-request data path is unchanged and continues to open each database with
SQLite `query_only`.

### Added — `sprint tasks -s` Short Alias

`sprint tasks` now accepts `-s` as a short alias for `--status`, matching the
documented contract and the short-flag convention used elsewhere in the CLI.

### Changed — Plain-Text Help Revised Across All Commands

Every plain-text help printer was reviewed and corrected for accuracy,
formatting, and completeness, so the on-screen help faithfully describes the
runtime behaviour. The corresponding help-related specifications were reconciled
with the behaviour verified empirically against the binary.

### Changed — AI Agent Contract Normalised and Completed

The `rmp --ai-help` contract was reworked for internal consistency and
completeness:

- Nested single-action commands are represented uniformly.
- Empty arrays are emitted as `[]` rather than `null`.
- Min-only ranges no longer carry a misleading `max: 0`.
- `--max-tasks` advertises its `1-10000` range.
- The `roadmap_flag` web exemption is documented (the conventions block notes
  that `web` does not require `-r`).
- `sprint tasks` exposes `-s`/`--status`.
- Failure examples are included for every command with non-zero exit codes.

### Fixed — Exit-Code Corrections to Match the Documented Contract

The following invocations previously returned an exit code that diverged from the
documented contract and are now corrected:

- **Invalid `--type` on list commands** — `backlog list` and `task list` now exit
  `6` (invalid data) instead of `1`.
- **Invalid `--status` on list commands** — `task list`, `sprint list`,
  `sprint tasks`, and `task stat` now exit `6` instead of `1`.
- **`sprint bottom` on a missing sprint** — now exits `4` (not found) instead of
  `6`.
- **`audit stats` empty-set timestamps** — the first/last timestamps are now
  emitted as JSON `null` rather than an empty string `""`.

### Dependencies

- **`GoGraph` bumped to `v0.4.0`**, with module dependencies refreshed
  (`go.mod`, `go.sum`).

### Internal

- New help-content unit suite `internal/commands/help_content_test.go`.
- New help/exit-code end-to-end contract suite
  `tests/test_44_help_and_exitcode_contract.py`.
- Extended AI-contract end-to-end suite `tests/test_30_aihelp_contract.py` to
  lock in the revised help text and contract invariants.

## Breaking Changes

None. The exit-code corrections align the runtime with the exit codes the AI
Agent Contract and the plain-text help already documented, so they restore the
promised behaviour rather than introducing a new one. No `rmp` command, flag, or
JSON success schema is removed or renamed, the on-disk graph format is unchanged,
and the database schema version is unchanged.

Callers that relied on the prior, undocumented exit codes (for example treating
an invalid `--status` as exit `1`) should update to the documented values
(invalid data is `6`, not found is `4`).

## Upgrade Notes

No manual migration steps are required. This release adds no SQLite migration of
its own and the database schema version is unchanged.

```sh
# Build from source
git fetch --tags
git checkout v1.12.0
go build -o ./bin/rmp ./cmd/rmp

# Verify
./bin/rmp --version
# Groadmap version 1.12.0

./bin/rmp --ai-help | grep binary_version
#   "binary_version": "1.12.0",
```

If your tooling parses exit codes from `task`/`sprint`/`backlog` list commands,
note the corrections above: invalid `--type`/`--status` now exit `6`, and
`sprint bottom` on a missing sprint exits `4`.

any benchmarked command. The Groadmap CLI command benchmarks therefore remain the
reference established in `v1.3.0` and are not re-measured here. The new web
features run only within the read-only `rmp web` server and add no per-command CLI
cost. Correctness is governed by the unit and E2E suites above rather than by a
throughput target.
